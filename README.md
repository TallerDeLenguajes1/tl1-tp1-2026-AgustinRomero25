Q: ¿Por qué es conveniente incluir el .gitignore?
A: Es fundamental para mantener el repositorio limpio, evitando que se suban archivos temporales, ejecutables, o carpetas de configuración del sistema/IDE que no forman parte del código fuente.

Q: ¿Cuándo se debe hacer?
A: Se debe crear preferentemente al inicio del proyecto, antes de realizar el primer commit, para asegurar que los archivos no deseados nunca sean rastreados por Git.

Q: ¿Cómo se debe configurar?
A: Se crea un archivo llamado ".gitignore" en la raiz del proyecto y se listan nombres de archivos, extensiones o carpetas que se deseen excluir.
